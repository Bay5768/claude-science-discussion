# Claude Science 讨论观察页

这是一个围绕 **Claude Science** 的公开讨论观察页，聚合官方资料、英文社区、中文媒体/公众号、B站实测与部分登录后样本，用来判断外界如何理解 Anthropic 在 AI for Science 方向上的产品布局。

页面地址：

https://bay5768.github.io/claude-science-discussion/

## 这个页面看什么

这不是 Claude Science 的功能复述页，而是一份轻量的舆情与产品观察：

- 外网是否把它看作“新模型”，还是更接近科研工作流入口。
- HN、Reddit 等社区认可哪些产品设计，例如可审计 artifact、HPC/数据库连接、Reviewer agent。
- 社区集中质疑什么，例如幻觉、论文垃圾化、生物安全、平台锁定和跨学科适用性。
- 中文侧如何传播、实测和上手，包括教程、视频、评论区问题与使用门槛。
- 这类科研工作台形态对后续 AI/Agent 平台设计有什么启发。

## 内容结构

- 一句话判断：压缩当前外部讨论的核心结论。
- 主要功能与科学任务：梳理 Claude Science 官方定位和任务范围。
- 工作流截图：用官方图、Docs 截图、B站实测图串起从任务输入到 artifact 输出的流程。
- Claude 在 Science 的布局时间线：记录 Anthropic 在生命科学、科研工作台、项目资助上的连续动作。
- 讨论主题与来源证据：把观点、代表声音和可点击来源放在一起。
- 教程与截图合集：单独收纳中文侧教程、视频实测和上手信号。
- 观察建议：给出后续应该持续跟踪的线索。
- 来源：列出官方、社区、媒体和中文侧样本。

## 覆盖边界

当前覆盖包括：

- Anthropic 官方发布、Claude Science 产品页与 Claude Docs。
- Hacker News 主帖与部分高信息密度评论。
- Reddit 公开搜索样本。
- 中文公开报道、公众号与转载线索。
- B站搜索与重点视频详情。
- 小红书登录后搜索样本中的部分评论信号。

弱覆盖包括：

- X 原生深帖和高互动讨论仍依赖平台内搜索或 API 补充。
- YouTube 搜索受 429/bot 校验影响，仅保留部分由 B站反查到的原始视频线索。
- 小红书长评、视频字幕/OCR 和连续热度变化仍需要后续补采。

## 更新方式

页面由本地静态文件生成，主文件是：

```text
index.html
assets/
```

更新时通常会做三件事：

1. 重新抓取 Claude Science 的公开讨论和教程/截图线索。
2. 更新 `index.html` 中的内容、来源、热度曲线和截图合集。
3. 发布到 GitHub Pages 与内部中转页。

## 本地预览

直接打开 `index.html` 即可预览；如果浏览器对本地文件策略较严格，也可以在目录中起一个静态服务：

```bash
python3 -m http.server 8080
```

然后访问：

```text
http://localhost:8080/
```

## 维护说明

- 不把非公开或登录态图片直接搬运到页面里。
- 对于不稳定来源，优先标注为“弱覆盖”或“登录后样本”。
- 有外部来源的截图卡片可以直接点击跳转。
- 页面是独立发布物，不依赖 Science Agent Base 或现有看板。
